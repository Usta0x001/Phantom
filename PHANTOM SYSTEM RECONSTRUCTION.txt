3. CONTROL FLOW
ReAct Execution Loop (BaseAgent.agent_loop)
# Pseudocode derived from base_agent.py lines 246-514
async def agent_loop(task: str) -> dict:
    await _initialize_sandbox_and_state(task)
    
    while True:
        # 1. Check stop conditions
        if _force_stop: await _enter_waiting_state(was_cancelled=True); continue
        if state.should_stop(): return state.final_result
        
        # 2. Increment iteration
        state.increment_iteration()
        
        # 3. Periodic status injection (every 10 iterations OR >50 messages)
        if iteration % 10 == 0 or message_count > 50:
            status = get_scan_status()
            state.add_message("user", format_scan_status(status))
        
        # 4. Periodic ledger injection (every N iterations, configurable)
        if iteration % LEDGER_INJECT_EVERY == 0:
            summary = hypothesis_ledger.to_prompt_summary(status_filter=["open", "testing"])
            state.add_message("user", summary)
        
        # 5. Periodic coverage injection (every 15 iterations, configurable)
        if iteration % COVERAGE_INJECT_EVERY == 0:
            summary = coverage_tracker.to_prompt_summary(max_items=15)
            state.add_message("user", summary)
        
        # 6. Periodic correlation injection (every 20 iterations, configurable)
        if iteration % CORRELATION_INJECT_EVERY == 0:
            summary = correlation_engine.to_prompt_summary(max_items=10)
            state.add_message("user", summary)
        
        # 7. Phase-gate warning at 85% iterations
        if iteration == 0.85 * max_iterations:
            state.add_message("user", FINAL_WARNING_MESSAGE)
        
        # 8. LLM inference
        async for response in llm.generate(state.get_conversation_history()):
            final_response = response
        
        # 9. Parse and execute actions
        actions = parse_tool_invocations(final_response)
        should_finish = await _execute_actions(actions, tracer)
        
        # 10. Checkpoint save (periodic)
        _maybe_save_checkpoint(tracer)
        
        # 11. Check completion
        if should_finish: return state.final_result

Termination Conditions (base_agent.py, state.py)
1. Completion: Agent calls finish_scan or agent_finish tool → state.set_completed()
2. User Stop: _force_stop = True → stop_requested = True
3. Iteration Limit: iteration >= max_iterations (default 300)
4. No-Action Stall: 8 consecutive iterations with no actions + non_interactive=True
5. Rate Limit Abort: 10 consecutive rate limit hits → abort with error
6. Unhandled Exception: Runtime/Value/TypeError in non-interactive mode

Summary
Phantom is a bounded autonomous security orchestration system with the following verified properties:
1. ReAct-based control with explicit state management
2. Dual-anchor memory (message-level + state-level) for high-signal preservation
3. Exact-match hypothesis deduplication (not semantic)
4. Thread-safe shared state for multi-agent coordination
5. Sandboxed tool execution (when enabled)
6. Periodic status reinjection (every 10 iterations by default)
The system makes no claims of completeness or universal detection - it is explicitly framed as an "orchestration layer for evidence accumulation" with documented failure modes.

## FORMALIZATION — CORE MECHANISMS
### 1. CVE Template Gap
**Definitions:**
- $T$ = template set (finite set of signature patterns)
- $V$ = vulnerability space (set of all possible vulnerability instances)
**Formal Condition for Detection:**
$$\text{detect}(v, T) = \begin{cases} 1 & \text{if } \exists t \in T : \text{matches}(t, v) \\ 0 & \text{otherwise} \end{cases}$$
**Scope Limitation (Template-Based Systems Only):**
$$\forall v \in V : v \notin T \implies \text{detect}(v, T) = 0$$
The epistemic ceiling: template-based detection is bounded by $|T|$; no orchestration improvement can overcome this structural impossibility.

2. Dual-Anchor Memory
Definitions:
- $M$ = message history (ordered list of conversation messages)
- $A$ = anchor set (high-signal items surviving compression)
- $C(M)$ = compression function (summarization operator)
Invariant:
$$A \subseteq C(M) \cup A_s$$
where $A_s$ = state-level anchors (finding_anchors).
Anchor Types:
Level	Location	Mechanism
Message-level	memory_compressor.py:676-680	Keyword matching via _ANCHOR_KEYWORDS_PATTERN
State-level	state.py:55-64	finding_anchors list re-injected before every LLM call
Failure Cases:
- Paraphrased findings (no keyword match) → lost
- Anchor keyword false negatives → critical context dropped
- Semantic drift in summarization → lossy compression despite anchors

3. HypothesisLedger
Definitions:
- Set $H \subseteq S \times C$ where:
  - $S$ = attack surface strings (e.g., /api/login::username)
  - $C$ = vulnerability class strings (e.g., sqli, xss)
Membership Function:
$$\mu(h, H) = \begin{cases} 1 & \text{if } \exists h' \in H : h.\text{surface} = h'.\text{surface} \land h.\text{vuln\_class} = h'.\text{vuln\_class} \\ 0 & \text{otherwise} \end{cases}$$
Deduplication Rule (Exact Match):
$$\text{add}(s, c) = \begin{cases} h'.\text{id} & \text{if } \exists h' \in H : h'.\text{surface} = s \land h'.\text{vuln\_class} = c \\ \text{new\_id} & \text{otherwise} \end{cases}$$
Limitation (No Semantic Similarity):
- /api/user/1 and /api/user/2 treated as distinct
- No equivalence relation $\sim$ on surfaces; only equality $=$

4. ReAct Loop
State Machine Definition:
$$S_t = (M_t, A_t, H_t, C_t, O_t, I_t)$$
where:
- $M_t$ = message history at iteration $t$
- $A_t$ = anchor set
- $H_t$ = hypothesis ledger state
- $C_t$ = coverage tracker state
- $O_t$ = observation set
- $I_t$ = iteration counter
Transition Function:
$$\delta : S_t \times \text{Action} \rightarrow S_{t+1}$$
Formal Loop:
$$\text{while } \neg \text{should\_stop}(S_t) :$$
$$\quad O_t \leftarrow \text{observe}(S_t)$$

Termination Conditions:

System Limitations and Failure Modes
1. Memory Failures
Anchor Keyword Miss
- Anchors extracted via regex match against 50+ hardcoded keywords (_ANCHOR_KEYWORDS_PATTERN in memory_compressor.py:121-124)
- Findings paraphrased without keyword matches are permanently lost during compression
- No mechanism to recover missed anchors post-compression
Compression Semantic Drift
- LLM-based summarization (_summarize_messages()) paraphrases content despite "PRESERVE EXACTLY" prompt instructions
- Critical technical details (exact payloads, response snippets) may be generalized or omitted
- Fallback summarization (_extract_message_text) truncates to 8000 chars with no semantic preservation
2. Reasoning Failures
Agent Inconsistency
- Each sub-agent owns independent LLM instance with isolated context window
- No consensus protocol for hypothesis status across agent boundaries
- Parent and child agents may hold contradictory beliefs about same surface
No Shared Reasoning State
- Shared components (HypothesisLedger, CoverageTracker) store factual state only
- Reasoning chains, decision rationales, and exploration strategies are not shared
- Cross-agent coordination limited to deduplication, not strategic alignment
3. Deduplication Limits
Exact Match Weakness
- Deduplication key: (surface, vuln_class) string equality only (hypothesis_ledger.py:81-83)
- Surfaces /api/user/1 and /api/user/2 treated as distinct hypotheses
- No semantic equivalence for URL patterns, parameterized endpoints, or structural similarities
- Duplicate testing occurs when surface strings differ syntactically but denote same attack vector
4. Execution Risks
Tool Hallucination vs Execution Mismatch
- No verification that executed tool matches LLM-intended tool
- Parameter injection validation disabled (_validate_tool_argument_injection() returns None unconditionally, executor.py:235-246)
- LLM may hallucinate tool capability; system executes literally without capability check
RBAC Bypass Risk
- RBAC module uses lazy import (executor.py:374-381)
- If RBAC module absent, all tools execute without access control
- No failure-closed default; absence of security module grants full access
5. Boundary Conditions
System Assumptions Break When:
Assumption	Failure Condition
Sandbox isolation	PHANTOM_SANDBOX_MODE=false (default) — tools execute on host OS
Rate limiting	_HTTP_TOOLS uses "terminal" but actual tool name is "terminal_execute" — stealth mode bypassed for primary attack tool
Budget warnings	_budget_warning_80_emitted declared at class level — shared across all LLM instances, fires once per process
Checkpoint integrity	HMAC-SHA256 provides integrity checking, not cryptographic verification
Thread safety	RLock protects individual operations, not cross-component consistency
Evidence validation	Heuristic keyword matching (_validate_evidence_quality()) — no correlation with actual vulnerability validity
Termination	No proof loop cannot diverge; infinite retry possible within iteration limit
Operational Boundary:
- Maximum work loss: 5 iterations (checkpoint interval)
- Context window ceiling: 80,000 tokens (MAX_CONTEXT_CEILING) regardless of model capacity
- Compression latency: 3–5 seconds per chunk, blocking LLM call pipeline
