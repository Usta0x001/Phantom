---
title: "CI/CD Integration"
description: "Run phantom in any CI/CD pipeline"
---

phantom runs in headless mode for automated pipelines.

## Headless Mode

Use the `-n` or `--non-interactive` flag:

```bash
phantom -n --target ./app --scan-mode quick
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | No vulnerabilities found |
| 1 | Execution error |
| 2 | Vulnerabilities found |

## GitLab CI

```yaml .gitlab-ci.yml
security-scan:
  image: docker:latest
  services:
    - docker:dind
  variables:
    phantom_LLM: $phantom_LLM
    LLM_API_KEY: $LLM_API_KEY
  script:
    - curl -sSL https://phantom.ai/install | bash
    - phantom -n -t ./ --scan-mode quick
```

## Jenkins

```groovy Jenkinsfile
pipeline {
    agent any
    environment {
        phantom_LLM = credentials('PHANTOM-llm')
        LLM_API_KEY = credentials('llm-api-key')
    }
    stages {
        stage('Security Scan') {
            steps {
                sh 'curl -sSL https://phantom.ai/install | bash'
                sh 'phantom -n -t ./ --scan-mode quick'
            }
        }
    }
}
```

## CircleCI

```yaml .circleci/config.yml
version: 2.1
jobs:
  security-scan:
    docker:
      - image: cimg/base:current
    steps:
      - checkout
      - setup_remote_docker
      - run:
          name: Install phantom
          command: curl -sSL https://phantom.ai/install | bash
      - run:
          name: Run Scan
          command: phantom -n -t ./ --scan-mode quick
```

<Note>
All CI platforms require Docker access. Ensure your runner has Docker available.
</Note>
